Linux Command Line

This page is a list of basic commands that you can execute from a Linux machine.

Linux Command and (common arguments) Description Explanation
ls (-lah) {directory name} ls is the command to list out contents of the current working directory The current working directory is the folder you are currently in. Equivalent to dir in Windows. l flag is for list, a is for list all, h is to show hidden files.
cd {directory} cd is the command to enter a directory. This command essentially allows you to enter or exit a folder, ../ to enter the previous path.
sudo {command} sudo is to execute any command as SuperUser. This command allows you to run any command that needs additional privileges that your current user does not have, for example, the ability to back up web directories.
pwd pwd is to allow you to see your current working directory. The current working directory is where you are executing the command from, eg /home/lee
cat {filename} cat allows you to print out a file into the current console. cat essentially prints out a file's content to the terminal, regardless if it is binary, it will try and print it out.
cp (-r) {source} {destination} cp allows you to copy files/folders, if your user has permission. cp essentially allows you to copy files by default, you are able to give absolute paths, like /home/lee/Desktop, or relative paths like ../Desktops. The -r flag is the recursive flag where you can copy all the contents of a folder.
mv {source} {destination} mv allows you to move files, if your user has permission. mv has the same syntax as cp, also able to give absolute or relative paths.
mkdir {foldername} mkdir allows you to create folders, if your user has permission. mkdir is able to create folders in folders that you have permission to. Also can create folder in other directories using absolute or relative path.
rm (-rfi) {target} rm allows you to remove folders/files (dangerous) Flags for rm includes r, f, i. r is recursive and goes into every folder to delete the contents. f is to force the delete. i is to ask user input for each file. This is dangerous as you are able to delete / (root) if you have privileges and destroys your machine. Targets includes files and folders and can contain relative or absolute paths.
