Linux Command Line

This page is a list of basic commands that you can execute from a Linux machine.

Linux Command and (common arguments) Description Explanation
ls (-lah) {directory name} ls is the command to list out contents of the current working directory The current working directory is the folder you are currently in. Equivalent to dir in Windows. l flag is for list, a is for list all, h is to show hidden files.
cd {directory} cd is the command to enter a directory. This command essentially allows you to enter or exit a folder, ../ to enter the previous path.
sudo {command} sudo is to execute any command as SuperUser. This command allows you to run any command that needs additional privileges that your current user does not have, for example, the ability to back up web directories.
pwd pwd is to allow you to see your current working directory. The current working directory is where you are executing the command from, eg /home/lee
cat {filename} cat allows you to print out a file into the current console. cat essentially prints out a file's content to the terminal, regardless if it is binary, it will try and print it out.
cp (-r) {source} {destination} cp allows you to copy files/folders, if your user has permission. cp essentially allows you to copy files by default, you are able to give absolute paths, like /home/lee/Desktop, or relative paths like ../Desktops. The -r flag is the recursive flag where you can copy all the contents of a folder.
mv {source} {destination} mv allows you to move files, if your user has permission. mv has the same syntax as cp, also able to give absolute or relative paths.
mkdir {foldername} mkdir allows you to create folders, if your user has permission. mkdir is able to create folders in folders that you have permission to. Also can create folder in other directories using absolute or relative path.
rm (-rfi) {target} rm allows you to remove folders/files (dangerous) Flags for rm includes r, f, i. r is recursive and goes into every folder to delete the contents. f is to force the delete. i is to ask user input for each file. This is dangerous as you are able to delete / (root) if you have privileges and destroys your machine. Targets includes files and folders and can contain relative or absolute paths.
touch {file} touch allows you to generate a blank file. As with the previous commands. You can only create a file if you have permission, also works with absolute or relative path.
find {directory} {expression} find allows you to find a file within a specific folder, using expressions. For find, you are able to use regex (more learning), or you can use flags like -name {filename} or -type {type} to specify what type of file/folder you are looking for, types includes but are not limited to h for hidden, d for directories.
grep {string} {file} OR {initial command} | grep {string} grep allows you to find a specific string in a file, or output of a command. There are 2 basic ways to use grep, to pipe the command into grep, or let grep have direct access to file. For example, you can do a direct access, like grep eth0 /var/log/syslog, or a pipe like dmesg | grep eth0. A pipe is this |, and this allows the piping of data to another application.
dmesg dmesg allows you to see the kernel log upon bootup, if you have privileges. A kernel log contains what the system basically verbosely outputs, for example a wireless module being unable to be loaded, or declaring how many CPU cores are present. Kernel logs have a lot of diagnostic data that is good for troubleshooting.
head {option} {file} OR {initial command} | head {option} head shows the first few lines of the file, or output of command. As with grep, it allows piping of commands. -n flag is to set how many lines are shown.
tail {option} {file} OR {initial command} | tail {option} tail shows the last few lines of the file, or output of command. Same syntax as head, as with grep, it allows piping of commands. -n flag is to set how many lines are shown.
chmod {attributes} {file} chmod allows you to change the attributes of a file. There are 2 main ways to change these for users, for example +x to add the execute bit, -w to remove the writable bit. There is also a numerical method, where each group has its bit, and you add the numbers together. For example, 4 is read, 2 is write and execute. So, using numerical, if I want to make a file +rwx, it will be chmod 777.
chown {user} {file} chown allows you to change the owner of a file/folder. To use this command, you have to be the superuser, or a higher privilege than the user you are changing ownership to.
ping {ip} ping allows you to test if an IP address is up, via sending ICMP packets and waiting for reply. ping only works if target IP address have not blocked ICMP packets.
wget {ip/path/to/file} wget allows you to retrieve a file from the internet. This command allows you to download a specific file from a server. For example, wget https://test.xyz/script.sh, to retrieve the script.sh file from test.xyz.
uname {options} uname prints out information about the kernel. uname has multiple flags, like -a to print out all information, -s for kernel name, like Darwin on MacOS, or Linux on Linux.
